# About 頁面內容重新設計 Design Spec

**Goal:** 將 About 頁面從制式模板風格改為敘事型個人介紹，面向潛在雇主/合作者，語氣專業但帶有個性。

**Scope:** 僅修改 `src/pages/about.astro` 的內容，不改動 layout 或樣式結構。

---

## 設計決策

- **讀者：** 潛在雇主與合作者
- **定位：** AI-augmented 開發者，從痛點出發造東西的人
- **語氣：** 專業但平易近人，帶點輕鬆個性
- **結構：** 純敘事型，不用條列式技能清單
- **聯絡：** GitHub + Email

## 移除的內容

- 技能條列清單（技能展現在作品和文章裡，不需要列出來）
- 部落格技術棧介紹（訪客不需要知道你用什麼框架）
- 假的 GitHub 連結

## 內容結構

### 1. 開場定位

一句話讓人知道你是誰、你的獨特之處。強調 AI 輔助開發不是補 code 的層次，而是重新想像開發流程。

### 2. 路徑故事

前端起步 → 公司碰後端 → 發現 AI 改變開發方式 → 投入探索 AI 協作流程。坦率不自貶，把轉折講清楚，收在 blog 是這條路的記錄。

### 3. Side Projects

用具體例子（記帳機器人、工時 CLI）帶出造東西的動機：解決自己的生活痛點。導流到 /projects 頁面。

### 4. 聯絡收尾

一句話邀請聯絡 + GitHub 和 Email 連結。

## 完整文案

```
Hi，我是 CCY。

軟體工程師，目前最感興趣的事情是搞清楚 AI 到底能怎麼幫工程師工作 — 不是讓 AI 幫你補 code 那種程度，而是從工作流程、協作方式到工具設計，重新想像開發這件事。

一開始是寫前端的，進了公司之後開始碰後端，但老實說兩邊都不算特別深。真正讓我著迷的轉折是 AI 工具開始變得夠好用 — 我發現與其糾結在每個框架的細節裡，不如把精力花在理解怎麼跟 AI 協作、怎麼設計讓 AI 發揮最大價值的流程。這個 blog 基本上就是這條路的記錄。

工作之外，我會做一些解決自己生活痛點的小東西 — 懶得手動記帳就做了一個 Discord 記帳機器人，覺得 Redmine 登工時太煩就寫了一個 CLI 幫自己自動化。完整的作品可以看 Projects 頁面。

如果你對 AI 輔助開發有興趣，或是單純想聊聊，歡迎寫信給我。

GitHub: https://github.com/cyl-jsl
Email: cyl.jsl@gmail.com
```

## 設計備註

- h1 維持英文 "About"，不改為中文。雖然 Projects 頁用「作品」，但 About 作為英文詞已足夠直覺，不需要強制統一。

## 技術實作

修改 `src/pages/about.astro`：

- 保留現有的 BaseLayout 包裹和 prose class
- 將 h1 保持為 "About"
- 用 `<p>` 標籤呈現各段文字
- 聯絡區塊用簡單的連結，不需要 icon
- Projects 連結使用 `<a href="/projects">`
